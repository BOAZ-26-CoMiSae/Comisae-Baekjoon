# 📚 자료구조: 큐(Queue) & 데크(Deque) 핵심 요약

## 1. 큐 (Queue)

먼저 들어온 데이터가 먼저 나가는 **선입선출(FIFO, First-In First-Out)** 구조입니다.

- **Rear (뒤)**: 데이터가 삽입되는 지점 (입구)
- **Front (앞)**: 데이터가 삭제되는 지점 (출구)
- **핵심 로직**:
  - 삭제 전에는 반드시 `isEmpty` (비어있는지) 검사
  - 추가 전에는 반드시 `isFull` (가득 찼는지) 검사

## 2. 선형 큐 (Linear Queue)

선형 배열을 사용하여 구현된 큐입니다.

### ⚠️ 배열의 한계 (공간 낭비)

선형 큐는 삭제된 후 데이터들을 앞으로 당기지 않고 화살표(`Front`)만 뒤로 이동합니다. 이 때문에 앞마당이 비어 있어도 입구(`Rear`)가 끝에 닿으면 더 이상 데이터를 넣을 수 없습니다.

**[동작 프로세스 예시]**

1. **데이터 삽입 (Enqueue)**
   - `[ 차1 | 차2 | 차3 | 빈칸 | 빈칸 ]`
   - `Front` = 0, `Rear` = 2
2. **데이터 삭제 (Dequeue)**
   - `[ 빈칸 | 빈칸 | 차3 | 빈칸 | 빈칸 ]`
   - 앞의 2칸은 **'버려진 땅'**이 되어 재사용 불가
3. **데이터 추가 삽입**
   - `[ 빈칸 | 빈칸 | 차3 | 차4 | 차5 ]`
   - `Rear`가 인덱스 4(끝)에 도달. **더 이상 삽입 불가!**

## 3. 원형 큐 (Circular Queue)

선형 큐의 단점을 보완한 구조로, 배열의 처음과 끝을 이어 도넛 모양으로 만든 것입니다.

- **특징**: `Front`와 `Rear`가 나머지 연산(`%`)을 통해 계속 회전합니다.
- **장점**: 버려지는 공간 없이 빈자리가 생기면 다시 돌아와서 **재활용**이 가능합니다.

## 4. 덱 (Deque, Double-Ended Queue)

Queue를 두 개 겹쳐놓은 것처럼 **양쪽 끝에서 입·출력이 모두 가능**한 만능 자료구조입니다.

- **특징**: `Rear`는 항상 마지막 요소를 가리키고 있으며, 스택과 큐의 기능을 동시에 수행할 수 있습니다.

### 🛠️ 주요 메서드 나열

| 작업 구분     | 예외 발생형 (Exception) | 값 반환형 (Safe) | 설명                   |
| :------------ | :---------------------- | :--------------- | :--------------------- |
| **앞쪽 삽입** | `addFirst()`            | `offerFirst()`   | 맨 앞에 데이터 추가    |
| **뒤쪽 삽입** | `addLast()`             | `offerLast()`    | 맨 뒤에 데이터 추가    |
| **앞쪽 삭제** | `removeFirst()`         | `pollFirst()`    | 맨 앞 데이터 삭제/반환 |
| **뒤쪽 삭제** | `removeLast()`          | `pollLast()`     | 맨 뒤 데이터 삭제/반환 |
| **앞쪽 확인** | `getFirst()`            | `peekFirst()`    | 맨 앞 데이터 확인      |
| **뒤쪽 확인** | `getLast()`             | `peekLast()`     | 맨 뒤 데이터 확인      |

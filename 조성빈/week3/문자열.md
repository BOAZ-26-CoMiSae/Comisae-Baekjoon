### 문자열 : 문자(Char)의 배열

- 배열 기반 → 내부적으로 char[] 배열과 유사한 구조
- 인덱싱 → 각 문자는 0부터 시작하는 인덱스를 가지며 O(1)의 시간 복잡도로 접근 가능
- 메모리 → Java 에서 char는 2 byte를 사용하므로 문자열의 길이에 비례해 메모리 증가

### Java의 문자열 관리 : String Pool

Java는 문자열을 효율적으로 관리하기 위해 String constant pool 이라는 특별한 영역을 힙 메모리 안에서 관리

- 재사용성
    - 똑같은 문자열을 여러번 선언하면 매번 새롭게 만들지 않고 풀에 있는 기존 주소로 연결
- 불변성
    - 이 풀을 유지하기 위해 String 은 한 번 생성하면 내용을 바꿀 수 없음.
    - 내용을 바꾸려하면 기존 주소 값을 바꾸는 것이 아니라 완전히 새로운 문자열을 메모리에 생성하고 변수가 그곳을 가리키게 만듦.

### 문자열 처리를 위한 핵심 자료구조

- StringBuilder (가변 배열)
    - 문자열을 자주 수정해야할 때 사용하는 가변 문자열 클래스
    - 내부적으로 여유 있는 공간이 있는 버퍼를 가지고 있어, 새로운 객체를 만들지 않고 값을 직접 수정

```python
public class StringBuilderExam {
    public static void main(String[] args) {
        StringBuilder sb = new StringBuilder();

        // 1. 문자열 추가 (append) - O(1)
        sb.append("Java");
        sb.append(" ").append("Coding");
        sb.append(" Test");

        // 2. 특정 위치에 삽입 (insert)
        sb.insert(4, " Is"); // "Java Is Coding Test"

        // 3. 문자열 뒤집기 (reverse) 
        sb.reverse();

        // 4. 최종 String으로 변환
        String result = sb.toString();
        System.out.println(result);
    }
}
```

- Trie (문자열 사전 탐색기) : Tree + Retrieval 의 합성어
    - 많은 문자열을 저장하고 특정 단어나 접두사가 포함되어있는지를 찾을 때 최적화된 트리 자료구조
    - 문자열의 길이만큼만 탐색
    - 노드 개수에 제한이 없음 
    (이진트리랑 다름! ca로 시작하는 단어가 5개있다면 a노드의 자식노드는 5개 이상이 생성된다는 뜻)

```python
import java.util.HashMap;
import java.util.Map;

class TrieNode {
    // 자식 노드들을 저장 (알파벳 배열 대신 Map을 쓰면 메모리가 효율적)
    Map<Character, TrieNode> children = new HashMap<>();
    boolean isEndOfWord; // 단어의 끝인지 표시
}

class Trie {
    TrieNode root = new TrieNode();

    // 단어 삽입
    void insert(String word) {
        TrieNode current = root;
        for (char c : word.toCharArray()) {
            // 없으면 새로 만들고, 있으면 이동
            current = current.children.computeIfAbsent(c, k -> new TrieNode());
        }
        current.isEndOfWord = true;
    }

    // 단어 검색
    boolean search(String word) {
        TrieNode current = root;
        for (char c : word.toCharArray()) {
            current = current.children.get(c);
            if (current == null) return false;
        }
        return current.isEndOfWord;
    }
}

public class TrieExam {
    public static void main(String[] args) {
        Trie trie = new Trie();
        trie.insert("apple");
        
        System.out.println(trie.search("apple")); // true
        System.out.println(trie.search("app"));   // false (접두사일 뿐 단어 끝이 아님)
    }
}
```

- HashMap (문자열 카운터 & 매퍼)
    - 문자열을 해시함수에 넣으면 고유 해시값 생성
    - 이 숫자를 배열의 인덱스로 삼아 값을 저장하거나 불러오기 가능

```python
import java.util.HashMap;

public class HashMapExam {
    public static void main(String[] args) {
        String[] participants = {"leo", "kiki", "eden", "leo"};
        HashMap<String, Integer> map = new HashMap<>();

        // 1. 빈도수 카운팅 (getOrDefault 사용이 핵심)
        for (String p : participants) {
            map.put(p, map.getOrDefault(p, 0) + 1); 
            //없으면 0을 가져와서 +1, 있으면 기존 값을 가져와서 +1
        }

        // 2. 특정 키 존재 여부 확인 - O(1)
        if (map.containsKey("leo")) {
            System.out.println("leo의 빈도: " + map.get("leo"));
        }

        // 3. 전체 순회
        for (String key : map.keySet()) {
            System.out.println(key + " : " + map.get(key));
        }
    }
}
```

# Hashmap과 Hashset 차이와 용도

- HashSet
    - 오직 값만 저장
    - 3052문제로 예를 들면 중볻을 허용하지 않은 상태에서 나머지들만 저장
- HashMap
    - key를 넣으면 연결된 데이터가 나옴
    - 3052문제로 예를 들면, (3,1) (5,2) 식으로 값에 따른 나머지를 묶음으로 저장

- `length`  사용시 에러 발생 ! `size`  사용해야함

| **구분** | **사용하는 키워드** | **의미** |
| --- | --- | --- |
| **배열 (Array)** | **`length`** | 처음 정해진 **전체 칸수** |
| **문자열 (String)** | **`length()`** | 문자의 개수 |
| **컬렉션 (Set, List)** | **`size()`** | 현재 실제로 **들어있는 데이터 개수** |

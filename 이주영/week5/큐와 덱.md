# 1. 개념 정리

---

## 큐(Queue)

### FIFO 구조

- 큐는 먼저 들어간 데이터가 가장 먼저 나오는 FIFO (First In First Out) 구조
- LIFO 구조인 스택(Stack)과 대비
- 주요 연산(offer, poll, peek)의 시간 복잡도가 모두 `O(1)`
- **코테 활용 예시**:
  1. BFS
  2. 위상 정렬 (Kahn's algorithm)
  3. 시뮬레이션 (작업 스케줄링, 프린터 큐 등)
  4. 0-1 BFS는 Deque 사용

## Java의 Queue 인터페이스

- `Queue`는 인터페이스이므로 구현체를 선택해서 사용
- 주요 메서드는 두 가지 형태로 제공됨

| 동작 | 예외 발생   | 특수 값 반환               |
| ---- | ----------- | -------------------------- |
| 삽입 | `add(e)`    | `offer(e)` → 실패 시 false |
| 제거 | `remove()`  | `poll()` → 비어있으면 null |
| 조회 | `element()` | `peek()` → 비어있으면 null |

- 코테에서는 예외가 발생하지 않는 `offer` / `poll` / `peek`가 더 안전

## 코테에서의 Queue 구현체

- 주요 구현체로 `LinkedList`, `ArrayDeque`, `PriorityQueue`가 있음
- 일반 큐 용도로는 **`ArrayDeque`** 권장
  - circular array로 구현되어 `LinkedList`보다 일반적으로 빠름
  - 노드 객체 생성 오버헤드가 없음
  - null 원소는 허용하지 않음
    - `LinkedList`와 다른 점
    - 코테에선 보통 문제 없음
- `LinkedList`도 자주 쓰이지만 성능상 `ArrayDeque`이 더 유리함
  - 접근 성능 좋음, 연산 속도 빠름, 더 적은 메모리 사용, 자바 공식 문서 추천 등
  - https://velog.io/@newdana01/Java-%ED%81%90-%EA%B5%AC%ED%98%84%EC%8B%9C-ArrayDeque%EC%99%80-LinkedList-%EC%84%B1%EB%8A%A5-%EC%B0%A8%EC%9D%B4-Deque-Queue-%EC%9D%B8%ED%84%B0%ED%8E%98%EC%9D%B4%EC%8A%A4

## 덱(Deque)

- Double Ended Queue의 줄임말, "덱"이라고 읽음
- 양 끝에서 삽입/삭제가 가능한 자료구조 → 큐로도, 스택으로도 쓸 수 있음
- 코테에서는 `ArrayDeque` 사용

### 코테 활용 예시

- 슬라이딩 윈도우 최댓값/최솟값 (monotonic deque)
- 0-1 BFS (가중치가 0 또는 1인 그래프 최단 경로)
- 양방향 시뮬레이션 (예: 회전하는 큐 문제)

---

## 우선순위 큐(PriorityQueue)

- 우선순위가 높은(기본은 작은) 원소가 먼저 나오는 큐
- 내부적으로 이진 힙(binary heap)으로 구현됨
- 시간 복잡도
  - 삽입(`offer`), 삭제(`poll`): `O(log n)`
  - 최솟값 조회(`peek`): `O(1)`
- 기본은 최소 힙(min-heap), `Comparator`로 최대 힙이나 커스텀 정렬 기준 지정 가능
  - 자세한 구현 및 예제 → https://cjh5414.github.io/priorityqueue/
- null 원소 허용하지 않음

### 코테 활용 예시

1. 다익스트라(Dijkstra) 알고리즘 (가장 빈번)
2. 프림(Prim) 알고리즘 (MST)
3. K번째 큰/작은 원소 찾기
4. 작업 스케줄링, 허프만 코딩

### 7. BFS 패턴

```java
import java.util.*;

public class BFS {
    public static void main(String[] args) {
        int n = 5;
        List<List<Integer>> graph = new ArrayList<>();
        for (int i = 0; i <= n; i++) graph.add(new ArrayList<>());
        // ... 간선 추가 graph.get(u).add(v);

        boolean[] visited = new boolean[n + 1];
        Queue<Integer> queue = new ArrayDeque<>();

        queue.offer(1);
        visited[1] = true;

        while (!queue.isEmpty()) {
            int cur = queue.poll();
            // 현재 노드 처리 로직

            for (int next : graph.get(cur)) {
                if (!visited[next]) {
                    visited[next] = true;
                    queue.offer(next);
                }
            }
        }
    }
}
```

## 덱(Deque) — ArrayDeque

### 1. 선언

```java
import java.util.ArrayDeque;
import java.util.Deque;

Deque<Integer> deque = new ArrayDeque<>();
```

### 2. offerFirst(e) / offerLast(e) : 양쪽 삽입

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Deque<Integer> deque = new ArrayDeque<>();

        deque.offerLast(20);  // [20]
        deque.offerLast(30);  // [20, 30]
        deque.offerFirst(10); // [10, 20, 30]

        System.out.println(deque); // [10, 20, 30]
    }
}
```

### 5. 슬라이딩 윈도우 최댓값 패턴 (Monotonic Deque)

- 윈도우 크기 `k`를 유지하며 매 윈도우의 최댓값을 `O(n)`에 구함
- 덱에 **인덱스**를 저장하고, 값이 감소하는 단조성을 유지

```java
import java.util.*;

public class SlidingWindowMax {
    public static int[] maxInWindow(int[] arr, int k) {
        int[] result = new int[arr.length - k + 1];
        Deque<Integer> deque = new ArrayDeque<>(); // 인덱스 저장

        for (int i = 0; i < arr.length; i++) {
            // 윈도우 범위를 벗어난 인덱스 제거
            while (!deque.isEmpty() && deque.peekFirst() <= i - k) {
                deque.pollFirst();
            }
            // 현재 값보다 작거나 같은 뒤쪽 인덱스들 제거 (감소 단조)
            while (!deque.isEmpty() && arr[deque.peekLast()] < arr[i]) {
                deque.pollLast();
            }
            deque.offerLast(i);

            if (i >= k - 1) {
                result[i - k + 1] = arr[deque.peekFirst()];
            }
        }
        return result;
    }
}
```

## 우선순위 큐(PriorityQueue)

### 1. 선언 — 기본 최소 힙

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        PriorityQueue<Integer> pq = new PriorityQueue<>();

        pq.offer(30);
        pq.offer(10);
        pq.offer(20);

        System.out.println(pq.poll()); // 10
        System.out.println(pq.poll()); // 20
        System.out.println(pq.poll()); // 30
    }
}
```

### 2. 최대 힙

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        // 방법 1: Comparator.reverseOrder()
        PriorityQueue<Integer> pq = new PriorityQueue<>(Comparator.reverseOrder());

        // 방법 2: 람다
        // PriorityQueue<Integer> pq = new PriorityQueue<>((a, b) -> b - a);

        pq.offer(10);
        pq.offer(30);
        pq.offer(20);

        System.out.println(pq.poll()); // 20
        System.out.println(pq.poll()); // 10
    }
}
```

### 3. 커스텀 객체 정렬

- 다익스트라처럼 `[거리, 노드]` 같은 형태로 자주 사용

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        // [거리, 노드] 배열을 거리 오름차순으로
        PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[0] - b[0]);

        pq.offer(new int[]{5, 1});
        pq.offer(new int[]{2, 3});
        pq.offer(new int[]{8, 2});

        int[] top = pq.poll();
        System.out.println(top[0] + " " + top[1]); // 2 3
    }
}
```

### 4. offer / poll / peek / size / isEmpty

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        PriorityQueue<Integer> pq = new PriorityQueue<>();

        pq.offer(30);
        pq.offer(10);
        pq.offer(20);

        System.out.println(pq.peek());    // 10
        System.out.println(pq.size());    // 3
        System.out.println(pq.poll());    // 10
        System.out.println(pq.isEmpty()); // false
    }
}
```

### 5. 다익스트라 패턴

```java
import java.util.*;

public class Dijkstra {
    public static int[] dijkstra(int n, List<List<int[]>> graph, int start) {
        int[] dist = new int[n + 1];
        Arrays.fill(dist, Integer.MAX_VALUE);
        dist[start] = 0;

        // [거리, 노드] - 거리 오름차순
        PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[0] - b[0]);
        pq.offer(new int[]{0, start});

        while (!pq.isEmpty()) {
            int[] cur = pq.poll();
            int d = cur[0], node = cur[1];

            // 이미 더 짧은 경로로 처리된 경우 스킵
            if (d > dist[node]) continue;

            for (int[] next : graph.get(node)) {
                int nNode = next[0], nCost = next[1];
                if (dist[node] + nCost < dist[nNode]) {
                    dist[nNode] = dist[node] + nCost;
                    pq.offer(new int[]{dist[nNode], nNode});
                }
            }
        }
        return dist;
    }
}
```
